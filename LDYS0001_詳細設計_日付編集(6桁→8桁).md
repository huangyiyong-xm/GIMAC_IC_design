## 1. 処理概要

### 1.1. 機能概要

引数より日付編集：6桁→8桁（yyMMdd→yyyyMMdd）
補足
日付変換を年の下二桁が80年以上の時年の頭二桁を19,20にくっつける処理
2100年には対応できなくなるため修正が必要になる。しかし、今回はこのままで進める

## 2. 詳細処理

### 2.1. 引数の取得とチェック

- 引数.日付が　ブランク　又は　 NULL 　の場合、エラーメッセージを出力し処理終了。
  - エラーコード : E.LHP100XX
  - エラーメッセージ : 'The argument ps_date_yymmdd is blank or NULL.'
    - (引数.日付 が　ブランク　又は　 NULL)
  - エラー位置 : '引数.日付'

### 2.2. 初期処理

利用する変数を初期化する。

| No. | 変数論理名 | 初期化設定値 |
| :-: | ---------- | ------------ |
|  1  | 変数.日付  | スペース     |

### 2.3. 主処理

### 2.3.1. 日付編集(6桁→8桁)

- 引数.日付（６桁） = '999999'の場合
  変数.日付 = '999999'
  -それ以外の場合
  - 引数.日付（６桁） = ''の場合
    変数.日付 = ''
  - それ以外の場合
    - SUBSTR(引数.日付（６桁）, 1, 2) > '80'の場合
      変数.日付 = '19' || 変数.日付
    - それ以外の場合
      変数.日付 = '20' || 変数.日付

### 2.4. 終了処理

取得した日付を戻り値に設定する。

| 戻り値論理名     | 設定値    |
| ---------------- | --------- |
| 処理ステータス   | 0         |
| SQL コード       | スペース  |
| エラーコード     | スペース  |
| エラーメッセージ | スペース  |
| エラー位置       | スペース  |
| 日付（8桁）      | 変数.日付 |

## 3. 補足説明

### 3.1. 戻り値について

- ステータスについて
  - 0 : Normal End
  - -1 : Abnormal End
  - -2 : PGM Error

### 3.2. エラー発生時の対応について

- SQLエラーが発生した場合、エラーログを出力して処理終了する
